按功能审计

安装问题

  • 安装完无验证导致重装

  • 验证后跳转,无exit

  • 步骤验证问题,有额外验证,或在其他if里面,导致重装

  • 变量覆盖导致重装,覆盖掉文件地址变量,导致file_exists 判断未false

  • 安装完重命名未install.php.bak 结合apache解析漏洞导致重装

    重装的利用

    • 写数据库文件,若是双引号,写入{@phpinfo()},代码执行
    • 未过滤,直接写shell
    • 尝试数据库外链,登录管理员
    • 利用表前缀,注入

变量覆盖

  • 导致变量覆盖的函数,伪全局变量覆盖问题

  • 利用点

    • 变量使用前未初始化,或者初始化在变量覆盖语句之前,导致文件包含或者注入等

  • 伪全局变量

    • 未阻止超全局变量的注册

      • 有上传功能,且用copy负责文件,覆盖

    先看有没有报错语句,有的话,报错注入

没有就找回显示,没有回显就盲注

  • 全局没有做addslashes的

  • 找未addslashes 的变量

  • 全局做addslashes

    • 宽字节注入

      • Set client gbk ,或者Set names gbk 导致宽字节注入
      • 转换字符集造成的宽字节注入
        • 錦 从UTF8 转成 GBK之后成了 %e5%5c 74,cms对GET POST COOKIE …… 都做了addslashes 所以’ 转义后为\’ ->%5C,錦’转意后为 %e5%5c%5c’ 两个\ 则单引号出来,导致引号逃逸

    • 解码导致

      • 解码的 例如urldecode base64_decode,带人查询导致绕过全局过滤导致注入

    • 变量覆盖

      • 覆盖掉表前缀导致注入,不用逃逸单引号

    • Replace 造成

      • 一些cms中 总有一些逗比过滤函数

        会把’ 啥的 replace 成空,导致注入

    • server

      • 未过滤server,或者过滤不严格导致
      • QUERY_STRING X_FORWARDED_FOR CLIENT_IP HTTP_HOST ACCEPT_LANGUAGE 很多。

    • file 注入,文件名,后缀入库导致注入

    • 未初始化

      • 一些cms引入伪全局机制,页面开始未初始化某些变量,导致可以覆盖,导致注入

    • 数组的key

      • 未过滤数组 的key导致注入

    • offset

      • 本意提交a为数组,若提交a为字符串可能导致截取字符,逃逸导致注入

    • 第三方插件导致

    • 数字型

    • 二次注入

    • 查询中Key可控

    • striplashes

    • 截取字符导致注入

php 安全问题

弱类型比较

==的弱类型比较

in_array 弱类型比较

In_array 若不加第三个参数,则默认进行弱类型比较,

可能造成数组里为数字,白名单验证绕过,导致注入

不安全的函数

mail 导致的命令执行

如果mail的第5个参数可控制,可导致 代码执行,文件读写,写入等问题

见PHP mail()函数漏洞总结

特殊情况,远程代码执行

在判断邮箱时忽略$email内双引号包裹 的内容,可通过重叠单引号而让filter_var认为是双引号

详情见:https://stackoverflow.com/questions/19220158/php-filter-validate-email-does-not-work-correctly

PHP的 mail() 函数在底层实现中，调用了 escapeshellcmd() 函数，对用户输入的邮箱地址进行检测,

若编写php代码时用了escapeshellarg,则可能多次处理造成单引号逃逸,

strpos

未找到会返回null,找到在第一个返回0,在弱类型情况下可能导致绕过

parse_str 函数缺陷

  • 未指定变量存储数组可能导致变量覆盖

  • 传入后会在进行url解码一次

  • url的host对@的解析与curl可能存在绕过

$_SERVER[‘PHP_SELF’]部分可控

若url为www.wendell.tong/index.php/admin/eval

会执行index.php ,php_self会拿到 index.php/admmin/eval

如果配合跳转,可能存在漏洞

md5()

传数组返回null,

md5(‘123’,true)

用原始二进制,而不是不以字符串返回

若带人数据库查询,可能导致注入

md5(128, true) 最后的结果带有反斜杠,可能造成单引号逃逸

ffifdyop 和129581926211651571912466741651878684928

会出现’or’af

可能变成万能密码

iconv 函数截断问题
